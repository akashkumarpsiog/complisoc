# DATA_MODEL.md

# Complisoc Data Model

Version: 1.0

---

# Purpose

Defines the canonical domain entities, relationships, lifecycle, and invariants used throughout Complisoc.

Data model goals:

* Traceability
* Auditability
* Reproducibility
* Explainability
* Historical preservation

Raw scanner findings are immutable.

All downstream processing creates derived records.

---

# Core Entities

```text
ScanRun

RawFinding

NormalizedFinding

ControlCatalog

ControlMapping

VerificationRecord

ReviewQueueItem

ComplianceReport

AuditBundle
```

# ScanRun

Represents a single execution of the compliance pipeline.

Relationships:

```text
ScanRun
    ├── RawFindings
    ├── NormalizedFindings
    ├── ControlMappings
    ├── ComplianceReports
    └── AuditBundles
```

Lifecycle:

```text
Created
    ↓
Running
    ↓
Completed

or

Failed
```

Invariants:

* Root entity of the platform
* Historical records must remain queryable

---

# RawFinding

Immutable finding generated by a scanner.

Sources:

* Trivy
* Checkov
* SonarQube
* Microsoft Defender

Relationships:

```text
RawFinding
    ↓
NormalizedFinding(s)
```

Invariants:

* Immutable
* Never modified
* Never overwritten
* Preserved as evidence

---

# NormalizedFinding

Canonical finding format used throughout the platform.

Purpose:

* Compliance mapping
* Verification
* Reporting
* Dashboard visualization

Relationships:

```text
RawFinding
    ↓
NormalizedFinding
    ↓
ControlMapping(s)
```

Required Attributes:

```text
finding_id
scan_run_id

scanner_name
finding_type

resource_type
resource_identifier

severity

title
description

timestamp
```

Invariants:

* Deterministically generated
* Traceable to source finding

---

# ControlCatalog

Reference library of compliance controls.

Frameworks:

* SOC2
* ISO27001

Relationships:

```text
ControlCatalog
    ↓
ControlMapping(s)
```

Required Attributes:

```text
framework_name
control_id
title
description
version
active_status
```

Invariants:

* Reference data only
* Versioned
* Loaded independently of scans

---

# ControlMapping

Compliance interpretation of a finding.

Primary output of the Compliance Intelligence Layer.

Relationships:

```text
NormalizedFinding
        ↓

ControlMapping

    ↙       ↘

Verification   Review
```

Required Attributes:

```text
mapping_id

finding_id
control_id

rationale

gemini_confidence

verification_status

final_confidence

mapping_status
```

Lifecycle:

```text
Generated
    ↓
Validated
    ↓
Verified
    ↓
Published

or

Manual Review

or

Rejected
```

Invariants:

* References a valid control
* Contains rationale
* Contains confidence score
* Never modifies findings

---

# VerificationRecord

Independent validation of a control mapping.

Generated by:

* Groq Verification Layer

Relationships:

```text
ControlMapping
        ↓
VerificationRecord
```

Required Attributes:

```text
verification_id

mapping_id

result

explanation

timestamp
```

Allowed Results:

```text
Agree
Disagree
```

Invariants:

* Immutable
* Part of audit evidence

---

# ReviewQueueItem

Represents a mapping requiring human review.

Entry Conditions:

* Confidence below threshold
* Validation failure
* Verification disagreement

Relationships:

```text
ReviewQueueItem
        ↓
ControlMapping
```

Allowed Outcomes:

```text
Approved
Rejected
```

Invariants:

* Required for uncertain mappings
* Published mappings bypass review only if threshold is met

---

# ComplianceReport

Governance artifact generated from validated compliance data.

Types:

```text
Engineering Report

Leadership Report
```

Relationships:

```text
ScanRun
    ↓
ComplianceReport
```

Invariants:

* Generated from validated mappings only
* Never generated from raw AI outputs

# AuditBundle

Exportable evidence package.

Contents:


Relationships:

```text
ScanRun
    ↓
```

Invariants:

* Traceable

# Compliance Lineage

Every published compliance decision must support:

```text
Published Mapping
        ↓

VerificationRecord
        ↓

ControlMapping
        ↓

NormalizedFinding
        ↓

RawFinding
        ↓

ScanRun
```

If lineage cannot be reconstructed, the mapping is invalid.

---

# Platform Invariants

The following rules are mandatory:

1. Raw findings are immutable.
2. Evidence records are immutable.
3. Normalization is deterministic.
4. Controls must exist in ControlCatalog.
5. Mappings require validation.
6. Mappings require verification.
7. Published mappings require lineage.
8. Reports use validated mappings only.
9. Audit bundles must be reproducible.
10. Historical records must remain queryable.
